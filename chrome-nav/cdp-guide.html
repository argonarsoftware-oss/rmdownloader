rmdownloader — CDP / Chrome Navigation & Regulation

Watch an AI's Chrome navigation live in the dashboard, and enforce per-site rules — block, warn, or replace — across every tab. For AIs you administer

Page: cdp.php (🧭 CDP) Tool: chnav.exe Rules file: blt.txt Protocol: Chrome DevTools Protocol
How it fits together: the 🧭 CDP page has no backend of its own — exactly like the DNS Manager. cdp.js sends PowerShell to the AI through the agent's exec op to launch chnav.exe and tail its log output back into the browser. chnav.exe attaches to Chrome over the Chrome DevTools Protocol (browser-level, so it covers every tab), prints navigation, and enforces the rules in blt.txt.

1 Build the exe

On a machine with Python 3 + PyInstaller (only needed to build; the exe then runs with no Python). Deps: pip install pyinstaller requests websocket-client.

cd chrome-nav
build.bat            :: -> dist\chnav.exe  (~10 MB, runs with no Python)

chnav.exe is a git-ignored build artifact — build it locally, don't commit.

2 Deploy it to the AI

Copy chnav.exe into one folder on each target AI — ideally next to the agent's own exe, which the page auto-detects. The rules file blt.txt is created in the same folder when you Save rules from the page.

:: example layout on the AI
C:\Users\Administrator\Desktop\chrome-nav\chnav.exe

3 Point the site at it (optional)

The folder auto-detects (running chnav → next to the agent → config fallback), so this is usually optional. To set the default folder and debug port, in website/config.php:

define('CDP_DIR', 'C:\\Users\\Administrator\\Desktop\\chrome-nav');
define('CDP_PORT', 9222);

4 Start it & watch navigation

Open the app, click 🧭 CDP in the top bar, pick the AI, then ▶ Start.

ControlWhat it does
▶ StartLaunches chnav.exe. It verifies the debug port first: if Chrome is already debugging it attaches; otherwise it kills the running Chrome and relaunches it with debugging (you can't add the port to a live Chrome).
■ Stop / ⟳ RestartStops / restarts the monitor.
requestsAlso log every request URL (DOC/req).
🔒 enforceAlways-on: re-seizes Chrome if it's closed and kills any Chrome that isn't the regulated instance, so the rules can't be escaped by closing it or opening a second window. (Chrome only — other browsers need OS policy.)
📡 FeedLive tail: NAV/SPA navigations and BLOCK/WARN/REPLACE/REDIRECT rule hits, newest first. Filter + auto-refresh.

5 Site rules — block, warn, replace

The 🛡 Site rules table is one row per domain. Saving writes blt.txt next to the exe; the monitor hot-reloads it (~2s) — no restart. *.x matches subdomains, and a bare domain also matches its subdomains.

ActionResult
BlockA 🚫 warning page replaces the site (the real load/redirect is stopped).
WarnA ⚠️ page with your custom message.
Replace withServes another site's page under the original address (URL bar stays — a spoof). An injected <base> makes the target's images/CSS/JS render correctly cross-origin; only its own API calls can still hit CORS.
Redirect toSends the tab to another site — the URL actually changes (e.g. gambling → https://phkarera.com/). Same-origin, so it renders cleanly. Best for "send this traffic elsewhere".

Saved blt.txt format (you can also edit it by hand):

facebook.com        replace https://www.youtube.com/
*.tiktok.com        block
news.example        warn  Back to work please
Replace caveat: a static page renders faithfully under the spoofed URL, but a live, script-driven site only half-works — once served under the original origin, the real site's scripts hit CORS/CSP/cookie-origin mismatches. Great for a convincing visual swap; not a working proxy of a live different site.

6 How it works under the hood

Launching Chrome with --remote-debugging-port exposes the Chrome DevTools Protocol. chnav.exe attaches at the browser level and auto-attaches to every tab, so rules apply everywhere. Notable details:

  • Block/warn are enforced on Page.frameNavigated by navigating the tab to a data: warning page — a real navigation that stops the in-flight load/redirect.
  • Replace keeps the URL by re-navigating the tab so the now-active Fetch domain catches the reload and serves the replacement response.
  • Launches with --remote-allow-origins=* (Chrome 111+ rejects the DevTools websocket with HTTP 403 otherwise) and --disk-cache-size=1 (so a cached page can't bypass the rules).
Scope & ethics: this seizes control of Chrome on the target AI and runs at the agent's privilege (SYSTEM if the agent auto-started elevated). Use it to regulate machines you own/administer. Serving fake content under a real URL to deceive other people is phishing/spoofing — don't.
