rmdownloader — VPS Setup Guide

Host the web file manager on your VPS and connect it to one or more Windows PCs. Manual deploy — no CI/auto-deploy

Domain: dos.argonar.co VPS: 172.104.186.245 Web root: /var/www/rmdownloader/website Repo: argonarsoftware-oss/rmdownloader
How it fits together: Agent.exe on each Windows PC reverse-connects out to this VPS over HTTPS and long-polls for commands — so there's no inbound port, no tunnel, no port-forwarding. The browser talks to the PHP site, the site queues a command, the agent picks it up, runs it, and posts the result back. On each PC you literally just run the agent.
✅ Done already: DNS A record (dos.argonar.co → 172.104.186.245) and packages (Apache, PHP, git, curl + rewrite/headers). Remaining steps below.

1 Clone the repo into /var/www

cd /var/www
git clone https://github.com/argonarsoftware-oss/rmdownloader.git

The repo is public, so this clone needs no login.

2 Create the config (with generated secrets)

This writes config.php (git-ignored) and prints the three secrets you need: the web password, the API key, and the PC's agent token.

cd /var/www/rmdownloader/website
cp config.sample.php config.php
TOKEN1=$(openssl rand -hex 24); APIKEY=$(openssl rand -hex 24); WEBPW=$(openssl rand -hex 6)
cat > config.php <<PHP
<?php
function rm_agents() {
    return array(
        'pc1' => array('name' => 'Main PC', 'token' => '$TOKEN1'),
    );
}
define('DATA_DIR', __DIR__ . '/data');
define('WEB_PASSWORD', '$WEBPW');
define('SESSION_TIMEOUT', 1800);
define('API_KEY', '$APIKEY');
PHP
echo "Web password: $WEBPW"; echo "API key: $APIKEY"; echo "PC1 token: $TOKEN1"
Each PC is just a name + token — no URL/port, because the agent dials out. Save the printed PC1 token for that machine's agent.conf.

3 Enable the Apache vhost

Name-based virtual host — it sits alongside your other sites, so don't disable the default site.

cp /var/www/rmdownloader/deploy/apache-vhost.conf /etc/apache2/sites-available/rmdownloader.conf
a2ensite rmdownloader
systemctl reload apache2

4 Set permissions & verify

Apache (www-data) must own the site and be able to write the data/ command queue:

chown -R www-data:www-data /var/www/rmdownloader/website
chmod -R 775 /var/www/rmdownloader/website/data

# Smoke test (works even before DNS/HTTPS); returns JSON, pc1 offline until the agent connects:
curl -s -H "Host: dos.argonar.co" "http://localhost/api.php?action=agents&key=$APIKEY"; echo

5 Add HTTPS (Let's Encrypt)

certbot --apache -d dos.argonar.co

Now https://dos.argonar.co serves the login page.

6 Build & run the agent (on each Windows PC)

No tunnel, no port-forwarding, no config file. The agent's server defaults to https://dos.argonar.co — you just pass the PC's token.

cd agent
build.bat                       :: produces Agent.exe (no installs needed)
Agent.exe <PC1 token from step 2>   :: test — prints "Connecting..."

Then install boot auto-start (elevated PowerShell) — the token is baked into the task, so it runs at every boot with no file to maintain:

powershell -ExecutionPolicy Bypass -File install-startup.ps1 -Token <PC1 token>

The task rmdownloaderAgent launches the agent at every boot as SYSTEM and restarts it if it exits. The PC shows 🟢 online in the web UI's picker within seconds. (A different server? add -Server https://your.host.)

7 Use it

Open https://dos.argonar.co, sign in, pick the PC, and browse its drives. Files download/upload straight through the VPS.

8 Updating later — manual

There is no auto-deploy. When you want the VPS to pick up new commits, pull by hand:

cd /var/www/rmdownloader
git pull
chown -R www-data:www-data website

Your config.php is git-ignored, so git pull never overwrites it.

Security checklist:
  • Use the generated random values for every agent token, WEB_PASSWORD, and API_KEY.
  • Keep HTTPS on (step 5) — the API key and tokens ride on it.
  • Optionally pass -Root C:\some\folder to install-startup.ps1 to sandbox the agent to one directory.
