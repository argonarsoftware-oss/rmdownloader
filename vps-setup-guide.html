rmdownloader — VPS Setup Guide

Host the web file manager on your VPS and connect it to one or more Windows PCs. Manual pull or GitHub [deploy] webhook

Domain: dos.argonar.co VPS: 172.104.186.245 Web root: /var/www/rmdownloader/website Repo: argonarsoftware-oss/rmdownloader
How it fits together: Agent.exe on each Windows PC reverse-connects out to this VPS over HTTPS and long-polls for commands — so there's no inbound port, no tunnel, no port-forwarding. The browser talks to the PHP site, the site queues a command, the agent picks it up, runs it, and posts the result back. On each PC you literally just run the agent.
✅ Done already: DNS, packages, repo clone, and config.php (secrets generated). Remaining steps below.

1 Enable the Apache vhost

Name-based virtual host — it sits alongside your other sites, so don't disable the default site.

cp /var/www/rmdownloader/deploy/apache-vhost.conf /etc/apache2/sites-available/rmdownloader.conf
a2ensite rmdownloader
systemctl reload apache2

2 Set permissions & verify

Apache (www-data) must own the site and be able to write the data/ command queue:

chown -R www-data:www-data /var/www/rmdownloader/website
chmod -R 775 /var/www/rmdownloader/website/data

# Smoke test — reads the key from config.php; returns JSON, pc1 offline until the agent connects:
cd /var/www/rmdownloader/website
APIKEY=$(grep -oP "API_KEY', '\K[^']+" config.php)
curl -s -H "Host: dos.argonar.co" "http://localhost/api.php?action=agents&key=$APIKEY"; echo

3 Add HTTPS (Let's Encrypt)

certbot --apache -d dos.argonar.co

Now https://dos.argonar.co serves the login page.

4 Build & run the agent (on each Windows PC)

No tunnel, no port-forwarding, no config file. The agent is windowless (runs silently in the background) and its server defaults to https://dos.argonar.co — you just pass the PC's token.

cd agent
build.bat                       :: produces Agent.exe (no installs needed)
Agent.exe <PC1 token from step 2>

On first run the agent self-installs a hidden boot task (rmdownloaderAgent) pointing at itself — run elevated for SYSTEM/boot, or normally for user/logon. It restarts on every boot, and the PC shows 🟢 online within seconds. (Skip with --no-autostart; remove with Agent.exe --uninstall.)

5 Use it

Open https://dos.argonar.co, sign in, pick the PC, and browse its drives. Files download/upload straight through the VPS.

The top bar also has two more tools that drive the same agent: 🌐 DNS — manage a network-wide TinyDNS server on the PC (records, blocklist, query log), and 🧭 CDP — watch Chrome navigation live and run local request mocks. Both need their helper exe (dnl.exe / chnav.exe) deployed on the PC.

6 Updating later — manual pull

To pull new commits onto the VPS by hand:

cd /var/www/rmdownloader
git pull
chown -R www-data:www-data website

Your config.php is git-ignored, so git pull never overwrites it.

7 Optional — auto-deploy on push (GitHub webhook)

Instead of pulling by hand, website/webhook-deploy.php can update the VPS automatically when you push — but only for commits whose message contains [deploy] (plain pushes are logged and skipped), so you control exactly when the live site changes. One-time server prep lets Apache's user run git in the repo:

chown -R www-data:www-data /var/www/rmdownloader
# mark the repo safe in the SYSTEM gitconfig (run as root, NOT `sudo -u www-data`):
git config --system --add safe.directory /var/www/rmdownloader

Set a secret in config.php ('' disables the webhook):

define('WEBHOOK_SECRET', 'a-long-random-value');

Then in GitHub → repo → Settings → Webhooks → Add webhook: Payload URL https://dos.argonar.co/webhook-deploy.php, content type application/json, the same secret, just the push event. Now git commit -m "fix: thing [deploy]" && git push runs git fetch + reset --hard origin/main on the VPS.

The hard reset only ever touches WEB_ROOT (it aborts unless that resolves to a git repo whose origin contains rmdownloader), and config.php is git-ignored so your secrets survive. Activity is logged to website/data/deploy.log.
Security checklist:
  • Use the generated random values for every agent token, WEB_PASSWORD, and API_KEY.
  • Keep HTTPS on (step 3) — the API key and tokens ride on it.
  • Optionally pass -Root C:\some\folder to install-startup.ps1 to sandbox the agent to one directory.
