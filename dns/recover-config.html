🛟 Recover records.txt / blocklist.txt

Rebuilds your custom DNS routes and blocks from queries.log after the old page bug deleted them. Run on the DNS machine. Click Copy, paste into PowerShell.
The recovery reads only queries.log (which survived). Step 1 is safe — it writes *.recovered.txt for you to review and changes nothing live.

1 · Preview the recoverysafe — review first

Writes records.recovered.txt and blocklist.recovered.txt in C:\Viewers. Open them and check they look right.

powershell -ExecutionPolicy Bypass -File C:\Users\Administrator\Desktop\rmdownloader\dns\recover-config.ps1 -Dir C:\ViewersCopy
→ prints "Recovered N route(s) and M blocked domain(s)" and writes the two .recovered.txt files.

2 · Apply for realwrites the live files

Replaces records.txt / blocklist.txt (backs up any existing to *.bak first). The DNS server hot-reloads them live — no restart.

powershell -ExecutionPolicy Bypass -File C:\Users\Administrator\Desktop\rmdownloader\dns\recover-config.ps1 -Dir C:\Viewers -ApplyCopy
→ "Applied to the live files." The DNS page editors will show the content after a hard-refresh.

3 · Verify

Check the files exist and the server reloaded them.

Get-Content C:\Viewers\records.txt; "----"; Get-Content C:\Viewers\blocklist.txtCopy
Then hard-refresh the DNS page with Ctrl+Shift+R — the editors load the recovered content and Save shows the green toast.
Note: wildcard rules (*.x) come back as the individual subdomains that were actually looked up (the log stores the queried name). Review and re-collapse them into *. rules if you like.
